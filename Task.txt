Сервис хранения GPS данных
1) Сервер должен запускаться как сервис Windows
2) Сервер должен содержать открытое (доступ без авторизации) и закрытое API (доступ только после авторизации)
3) Открытое API: 
  - считывание текущего время сервера
  - регистрация/смена пароля пользователя (в качестве имени пользователя использовать номер, генерация пароля для пользователя на стороне сервера)
4) Закрытое API: 
  - загрузка данных GPS на сервер (широта, долгота, время) по протоколам JSON, Protobuf
  - считывание данных GPS (широта, долгота, время) по номеру пользователя в заданном временном интервале по протоколам JSON, Protobuf
5) Загруженные на сервер GPS данные должны хранится в БД 

Консольный клиент для сервиса
1) При старте переходит в режим ожидания команд:
2) time - вывести текущее время сервера
3) start UsersCount StartDate - (UsersCount до 100) запускает имитацию UsersCount пользователей с датой начала StartDate (формат даты DD.MM.YYYY)
     - каждый пользователь запускается в своем потоке
     - каждый пользователь должен пройти автоматическую регистрацию на сервере
     - каждый пользователь посылает на сервер рандомные GPS координаты один раз в секунду 
       (начальные значения широты и долготы рандомные, время текущее, а все последующие небольшие рандомные смещения от предыдущего значения)
     - запуск повторной имитации останавливает предыдущую имитацию
4) stop - остановка имитации пользователей
5) history UserNumber FromDate ToData - считывание загруженных данных по пользователю с заданным номером за заданный интервал времени
     - формат даты DD.MM.YYYY
6) exit - выход
